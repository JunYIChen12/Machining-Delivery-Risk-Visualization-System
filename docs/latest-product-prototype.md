# 产品原型：机加工行业

> 由最新 Word 文档整理为 Markdown，便于 GitHub 检索、审阅和版本对比。Word 原件因当前连接工具限制暂未直接上传，已保留在本地项目包中。

## 背景

在机加工行业中，车间现场不是一条订单从头到尾顺着走，而是多个订单、多个成品、多个半成品规格同时在不同工序里交叉流转。

常见情况包括：

- 有的订单成品已经接近汇总，有的还卡在前段加工。
- 同一个订单里，有的半成品已经做完，有的还在排队，有的正在返工。
- 同一道工序不会一直做同一张订单，而是在不同订单、不同规格之间切换。
- 表面上设备和工人都很忙，但交付风险隐藏在订单进度不同步、成品流转不同步、零件加工不同步里。

## 示例产品

产品名称：液压快速换模系统25T。

固定半成品结构：

- 销轴
- 缸体
- 活塞
- 压板
- 基座

注意：不同订单可能是液压快速换模系统25T、30T、10T等。半成品名称和工艺结构固定，变化的是规格。

例如：

- 销轴规格：45*145
- 销轴规格：45*140
- 缸体规格：235*190*135
- 缸体规格：230*170*135

这意味着“半成品名称相同”不代表可以互相替代，系统必须按“半成品名称 + 规格”识别生产事实。

## 生产流程

生产计划员按照订单交付日期制定生产任务卡，然后下发任务卡给加工人员。

任务卡颗粒度：

- 销售订单
- 交付日期
- 单个成品所需半成品
- 规格
- 数量

加工人员确认原材料规格后，把原材料拿到对应设备加工。每完成一道工序就报工一次。

报工记录颗粒度：

- 原材料名称
- 规格
- 工序
- 数量

单个原材料走完整条工艺路线后，才算形成一个半成品加工结果。

## 真实生产情况

### 情况一：多订单交叉加工

例如销轴25T原材料规格45*145，完成第一道车床加工后，因为需要拉到别处加工，不能马上进入下一道工序。操作工会转而寻找其他订单所需原材料继续加工，例如订单B所需活塞规格85*87。

这会导致多个订单、多个半成品、多个规格在不同工序之间交叉推进。

### 情况二：半成品跨订单挪用

加工完的半成品原本服务A订单，但由于B订单只差这个半成品就能交付，现场可能把它挪去服务B订单。

例如压板25T原材料规格215*90*75，原本服务A订单，但因为B订单临近交付而被挪用。

重要约束：不管半成品最终是否被挪用，单位产能仍然要跟随工单任务数量体现出来。挪用只影响最终服务对象，不应该抹掉已经形成的生产事实。

## 核心目标

把混乱的机加工报工记录整理成：

- 半成品做到哪了
- 订单相关件做到哪了
- 半成品名称 + 规格形成了多少产能
- 哪些工序今天加工了哪些规格件

先让厂长和计划员看清生产事实。

## 产品定位

这款产品不是 MES 报工看板，也不是传统订单进度表，而是面向机加工多订单交叉生产场景的：

> 半成品产能池与订单相关加工进度可视化系统。

第一阶段只负责把生产事实摊清楚：

- 哪些订单关联了哪些工单
- 哪些工单对应哪些半成品、规格和工艺路线
- 每个半成品现在做到哪道工序
- 末道工序完成了多少
- 在制数量还有多少
- 某道工序今天加工的产出来自哪些订单和规格

系统不直接判断半成品最终服务哪张销售订单，不自动判断订单一定能不能交付，也不替厂长做挪用、配套和调度决策。

## 能力一：订单相关加工进度可视化

从销售订单往下穿透，查看订单对应的生产任务卡/MES工单、产品、半成品、规格、目标数量、工艺路线、当前工序、末道完工数量和最后报工时间。

打开一张订单时，不只是看到“有人在做”，而是看到该订单相关件分别做到哪里。

这里展示的是“订单相关加工事实”，不是最终交付结论。

## 能力二：半成品产能池可视化

按以下维度汇总分散报工记录：

- 半成品名称
- 规格
- 工艺路线
- 末道工序完工数量

形成半成品产能池。

这个产能池不是库存，也不是订单专属进度，而是基于可信报工链路推算出的当前加工能力。

## 能力三：自上而下追溯

支持链路：

销售订单 → 生产任务卡/MES工单 → 产品 → 半成品 → 规格 → 工艺路线 → 工序 → 报工记录

解决问题：一张订单相关的件到底有哪些、每个件做到哪一步、哪些报工记录支撑当前进度。

## 能力四：自下而上追溯

支持链路：

报工记录 → 工序 → 半成品 → 规格 → 工单 → 订单来源

解决问题：某道工序今天到底加工了哪些半成品、属于哪些工单、关联哪些订单。

## 能力五：复杂交叉生产状态可视化

系统必须同时展示：

- 订单维度的加工进度
- 半成品维度的产能池
- 工序维度的产出来源

从而把散落在 MES 报工里的碎片信息重新组织成可理解的生产画面。

## 能力边界

第一阶段系统只做事实展示和追溯：

- 不做自动齐套模拟
- 不判断半成品最终归属
- 不读取不可信库存
- 不输出“哪些订单一定能交”
- 不输出“今天必须优先做什么”的系统结论

系统价值是为厂长和计划员提供清晰、可信、可追溯的生产事实底座。
