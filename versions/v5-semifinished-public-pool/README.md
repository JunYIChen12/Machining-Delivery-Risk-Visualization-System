# v5：半成品公共池 MVP

这是当前最新产品定义版本。

## 核心变化

产品从“生产事实底座 / 规格产能池”进一步收敛为：

> 半成品公共池可视化看板。

第一版只回答两个问题：

1. 现在理论上有多少半成品生产能力？
2. 哪些订单需要这个半成品？

## 核心对象

半成品公共池不是库存池，也不是订单占用池。

它是基于可信 BOM、标准工艺路线和报工记录，按“半成品名称 + 规格”反推出来的理论生产事实池。

## 关键口径

理论完成量 = 所有必经工序报工数量的最小值。

例如压板工序进度为 `12 / 10 / 9`，理论完成量就是 `9`。

## 第一版不做

- 不读取库存
- 不判断订单占用
- 不判断真实可交付
- 不做自动排产
- 不做交付预测
- 不给出“应该给谁”的分配结论
